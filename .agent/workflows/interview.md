## Interview Workflow

Before starting a complex task (e.g., an audit, large feature, or architectural refactor), gather focused context from the user by asking concise, numbered questions. 

### Steps

1. Identify the **type of task** from the user's request.
2. Formulate **5–10 targeted questions** covering:
   - End goals and success criteria
   - What is in scope vs out of scope
   - Known problem areas or risks
   - Target users and user experience priorities
   - Constraints (time, budget, platforms, reversibility)
   - Any existing documentation or relevant datasets
3. Present all questions in a single message as a numbered list. Do NOT ask one question at a time.
4. Wait for the user's answers before proceeding.
5. Summarize the gathered context back to the user in 3–5 bullet points, then proceed with the task.
